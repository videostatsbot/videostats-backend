# Video Stats Bot

Проектная практика. Кейс 2. Агрегация статистики видео (PJ)
---

Командный проект хакатона 2 семестра магистратуры НИЯУ МИФИ.
<br>
Команда разработчиков: Аннамурадов Азатмурад, Мешко Егор, Ибрагимов Кямран, Цветков Александр, Арутюнян Сурен.
<br>
Представляет собой Java-приложение с интеграцией Telegram API для работы с ботом и поддержкой двух платформ видеохостингов: **YouTube** и **Rutube**.

## Основной функционал приложения

### Управление видео
- **Загрузка видео по ссылке** — автоматическое распознавание платформы (YouTube/Rutube) и добавление видео в базу
- **Ручное обновление статистики** — обновление данных просмотров для конкретного видео через карточку
- **Автоматическое обновление** — массовое обновление статистики всех загруженных видео одной командой

### Поиск и навигация
- **Поиск по названию** — поиск видео по ключевым словам в названии
- **Фильтрация по платформе** — отдельные списки для YouTube и Rutube
- **Пагинация** — удобная навигация по спискам видео и результатам поиска

### Статистика
- **Общая сводка** — статистика по всем видео (количество, суммарные просмотры)
- **История просмотров** — отслеживание динамики просмотров для каждого видео
- **Детальная карточка видео** — полная информация о видео с историей изменений

### Авторизация
- **Blacklist-авторизация** — доступ к боту имеют только пользователи, чьи Telegram ID указаны в конфигурации
- Проверка прав доступа при каждом запросе (сообщения и callback)

## База данных

Проект использует **PostgreSQL** со следующими таблицами:

- **users** — пользователи бота (Telegram ID, роль)
- **videos** — загруженные видео (платформа, URL, статистика)
- **video_stats_history** — история изменений статистики просмотров

Миграции управляются через **Flyway**.

## Конфигурация

Все настройки передаются через переменные окружения:

| Переменная | Описание                                       |
|------------|------------------------------------------------|
| `DATABASE_URL` | JDBC URL подключения к PostgreSQL              |
| `DATABASE_USER` | Пользователь базы данных                       |
| `DATABASE_PASSWORD` | Пароль базы данных                              |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram бота                             |
| `TELEGRAM_ALLOWED_USER_IDS` | Список разрешённых Telegram ID (через запятую)  |
| `YOUTUBE_API_KEY` | API ключ YouTube Data API                       |

## Команды ТГ бота

### Основные команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветственное сообщение и главное меню |
| `/help` | Справка по использованию бота |
| `/stats` | Показать общую статистику по всем видео |
| `/update` | Автоматически обновить статистику всех видео |
| `/list` | Показать список всех загруженных видео |
| `/list_youtube` | Показать список видео с YouTube |
| `/list_rutube` | Показать список видео с Rutube |
| `/search <запрос>` | Поиск видео по названию |
| `/delete <ID>` | Удалить видео по его ID |

### Команды через кнопки (Callback)

- **📋 Показать список** — навигация по всем видео
- **📺 YouTube / 🎬 Rutube** — фильтрация по платформе
- **🔄 Обновить всё** — массовое обновление статистики
- **🔍 Поиск** — интерактивный поиск с пагинацией
- **📊 Статистика** — просмотр общей сводки
- **ℹ️ Справка** — подробные инструкции по командам

### Карточка видео

При открытии карточки видео доступны действия:

| Действие | Описание |
|----------|----------|
| **Обновить** | Обновить статистику конкретного видео |
| **Удалить** | Удалить видео из базы |
| **Назад** | Вернуться к предыдущему экрану |

## Quick Start

### Предварительные требования

- Java 17+
- Docker и Docker Compose
- API-ключи:
    - Telegram Bot Token (можно получить у [@BotFather](https://t.me/BotFather) создав собственного бота)
    - YouTube Data API Key (можно получить в [Google Cloud Console](https://console.cloud.google.com/))

### Клонирование репозитория

```bash
git clone https://github.com/videostatsbot/videostats-backend.git
cd <project-directory>
```

### Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните файл `.env` необходимыми значениями:

```env
# База данных
DATABASE_URL=jdbc:postgresql://db:5432/tg_bot_video_stats
DATABASE_USER=postgres
DATABASE_PASSWORD=assword

# Telegram
TELEGRAM_BOT_TOKEN=your_token
TELEGRAM_ALLOWED_USER_IDS=id1,id2,id3

# YouTube API
YOUTUBE_API_KEY=your
```

> Узнать свой Telegram ID можно через [@userinfobot](https://t.me/userinfobot)

### Запуск через Docker Compose

```bash
docker-compose up --build -d
```

Проверка статуса контейнеров:

```bash
docker-compose ps
```

Просмотр логов приложения:

```bash
docker-compose logs -f app
```

### Тестирование бота

1. Откройте Telegram и найдите вашего бота по токену или имени
2. Отправьте команду `/start`
3. Добавьте видео, отправив ссылку на YouTube или Rutube:
   ```
   https://www.youtube.com/watch?v=VIDEO_ID
   ```
   или
   ```
   https://rutube.ru/video/VIDEO_ID/
   ```
4. Проверьте статистику командой `/stats`
5. Обновите данные командой `/update`

### Остановка приложения

```bash
docker-compose down
```
